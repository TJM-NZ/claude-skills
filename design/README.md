# Design

Design system and component architecture guidance for building consistent, accessible, and maintainable UI components.

## What It Does

Provides systematic design guidance for:

- **Component Architecture** - Atomic design, composition patterns, prop APIs
- **Design Tokens** - Color, typography, spacing, sizing systems
- **Accessibility** - WCAG compliance, keyboard navigation, screen readers
- **Responsive Design** - Mobile-first, breakpoints, fluid layouts
- **Design Patterns** - Common UI patterns and best practices
- **Component APIs** - Prop design, variants, controlled/uncontrolled

## When to Use

- Building new design system
- Improving existing component library
- Ensuring design consistency
- Accessibility improvements
- Component API design decisions

## Key Principles

### Component Architecture
- Start with primitives (Button, Input, Card)
- Compose complex components from primitives
- Single responsibility per component
- Controlled vs uncontrolled patterns

### Design Tokens
- Semantic naming (primary, secondary, not blue, red)
- Consistent spacing scale (4px, 8px, 16px, 24px, 32px...)
- Typography system (sizes, weights, line-heights)
- Color palette with accessibility in mind

### Accessibility
- Keyboard navigation (tab order, focus indicators)
- ARIA labels and roles where needed
- Color contrast ratios (4.5:1 text, 3:1 UI)
- Screen reader compatibility

## For Employers

Demonstrates design system thinking, component architecture, accessibility knowledge, and systematic approach to UI consistency.
